Now that the basic functionality is complete, it is time for a UX/UI overhaul, as the current design and UX is quite basic and not very user-friendly.

The application should have a clean but modern, intuitive and user-friendly interface which is a pleasure to see and use.
Teal as a color may remain if you think it fits well.
I would really like to see dark mode too.

Some functional improvements could be done too, like the "Deploying..." state is stuck and doesn't go away, only with a reload, resulting in disabled buttons.
We should also show a toast or similar notification when the deployment is successful or failed.
Also, if the user made configuration changes that needs deploy to take effect but didn't deploy, we should show a warning indicator somewhere, keeping it subtle, but noticeable.

We can get rid of the default JavaScript alerts and confirmations, and use our own custom ones.

We should have a consistent design language throughout the application.
Feel free to make any changes, redesigns and rearrangements you think would improve the user experience and visual design, we don't have to stick to the current design, arrangement or anything.

If you have any questions or need clarification, feel free to ask, but it is now mostly up to you. Take your time, it is a substantial task.
